## 1. 제품 개요

### 1.1 한 줄 정의
> "1on1 미팅 녹음 파일 하나로, 팀장은 팀원의 진짜 속마음을 알고 팀원은 자신의 성장을 자산으로 쌓는다."

### 1.2 핵심 문제
| 페르소나 | 현재 고통 |
|----------|-----------|
| 팀장 (리더) | 팀원이 서베이에서 "괜찮다"고 해도 실제로 괜찮은지 알 수 없다. 1on1이 형식적으로 흐른다. |
| 팀원 (멤버) | 1on1에서 한 말, 받은 피드백, 이룬 성과가 아무 기록 없이 휘발된다. 커리어를 증명할 수단이 없다. |

### 1.3 해결 방식
- 미팅 녹음 → STT → LLM 분석으로 **Honesty Gap(표면 답변 vs 실제 뉘앙스 차이)** 수치화
- 리더에게는 팀 건강 지표와 행동 처방전 제공
- 멤버에게는 성과·성장·고민이 자동 축적되는 커리어 자산 제공

## 2. 사용자 및 역할

| 역할 | 설명 | 주요 관심사 |
|------|------|-------------|
| **LEADER** (팀장) | 팀을 생성하고 1on1을 주관 | 팀 건강 지표, 위험 멤버 조기 감지, 약속 이행 |
| **MEMBER** (팀원) | 초대받아 팀에 참여, 1on1에 참석 | 내 성장 기록, 리더 약속 추적, 솔직한 피드백 수신 |

---

## 3. 핵심 개념 정의

| 용어 | 정의 |
|------|------|
| **Honesty Gap** | 사전 서베이 점수(X축) − AI 추론 점수(Y축). 클수록 표면과 속마음의 괴리가 큼 |
| **Safety Score** | 멤버의 행동(Initiative·Dissent·Vulnerability) 기반 심리적 안전감 지수 (0~100) |
| **Alignment Gap** | 서베이에서 원했던 주제(커리어 등)가 실제 대화에서 다뤄진 비율 차이 |
| **Talk Ratio** | 리더 발화 % / 멤버 발화 %. 권장 리더 40% / 멤버 60% |
| **사분면** | (서베이↑ AI↑)=안정, (서베이↑ AI↓)=조용한 위험, (서베이↓ AI↓)=명시적 위험, (서베이↓ AI↑)=보수적 응답 |
| **Career Event** | 대화에서 AI가 자동 추출한 성취·배움·블로커·제안 채택 이벤트 |
| **Promise** | 미팅 중 리더가 멤버에게 한 약속. 미이행 시 다음 1on1에 자동 알림 |

### 4.4 AI 분석 파이프라인

| 단계 | 작업 | 담당 |
|------|------|------|
| Step 1 (STT) | WebM → Whisper API → 전체 대화 텍스트 | SttAdapter |
| Step 2 (NLP) | 텍스트 → LLM → 행동 카운팅, AI 패턴, 블로커 키워드 추출 | LlmAdapter |
| Step 3 (SCORING) | survey_score + ai_score → Honesty Gap, Safety Score, Alignment Gap, Talk Ratio, 사분면 분류 | 내부 로직 |
| Step 4 (FEEDBACK) | 분석 결과 → LLM → 리더/멤버 피드백 + Next Action Plan + Career Event 생성 | LlmAdapter |

**LLM 출력 형식**: 항상 구조화된 JSON. 파싱 실패 시 1회 재시도 후 `FAILED` 처리.

**분석 중 로딩 UX용 위트 카피 (폴링 응답에 포함)**

| Step | 리더용 | 멤버용 |
|------|--------|--------|
| STT | "팀원이 '괜찮다'고 했을 때, 진짜 괜찮은 건지 확인 중..." | "오늘 내가 얼마나 솔직했는지 들여다보는 중..." |
| NLP | "행간의 한숨까지 읽는 중..." | "대화 속 나의 패턴을 분석 중..." |
| SCORING | "점수로 환산 불가한 감정을 억지로 환산 중..." | "오늘 대화의 진짜 무게를 재는 중..." |
| FEEDBACK | "어떻게 말해야 덜 아플지 고민 중..." | "다음엔 어떻게 말하면 좋을지 정리 중..." |

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 분석 응답 시간 | 업로드 후 202 즉시 반환. 분석 완료 목표 90초 이내 |
| 파일 크기 제한 | WebM 최대 25MB (Whisper API 제한) |
| 인증 만료 | AccessToken 1시간, RefreshToken 14일 |
| LLM 재시도 | JSON 파싱 실패 시 1회 자동 재시도 |
| 어댑터 교체 | STT·LLM 구현체를 @Profile로 무중단 교체 가능 |
| DB | PostgreSQL (Supabase). JSONB로 유연한 AI 결과 저장 |

## 7. 제외 범위 (Out of Scope)
